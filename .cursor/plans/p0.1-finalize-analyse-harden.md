# P0.1 — Finalize/analyse harden (restant)

## Context

Twee UX-modi:

| Pad | Gedrag bij AI-fout |
|-----|-------------------|
| Toolbar AI | Non-blocking, vriendelijke fallback |
| Opslaan / finalize | Blocking — geen analyse-review zonder `entry_analyses` |

TwinWord-falen is geen volledige analyse-fout; alleen LLM-fouten blokkeren finalize.

## Geïmplementeerd

- `lib/ai/openai-errors.ts` — gedeelde `mapOpenAiError` voor `agent` en `analysis`
- `lib/ai/agent.ts` — gebruikt gedeelde mapper
- `lib/ai/analyze-entry.ts` — try/catch, validatie kernvelden, TwinWord best-effort
- `components/journal/WritingArea.tsx` — aparte `finalizeError` state
- `components/journal/WritingToolbar.tsx` — toont finalize-fout met rode melding

## Acceptatiecriteria

1. OpenAI 429/5xx bij Opslaan → `{ error }`, gebruiker blijft op schrijfscherm
2. TwinWord-fout → finalize slaagt
3. Ontbrekende kernvelden → finalize faalt
4. Coach-acties blijven non-blocking
5. Rate-limit copy bij analyse is specifiek (niet “ga door met schrijven”)
